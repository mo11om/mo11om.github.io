The Problem

Most networking tutorials treat DNS as a black box: call getaddrinfo(), receive an IP. This project goes the other direction — the server is the resolver. It receives raw DNS queries over UDP, parses binary wire format by hand, looks up records in zone files, and only calls upstream when it can’t answer authoritatively.

The engineering challenge is binary protocol implementation: DNS packets have no delimiters, use compressed label encoding, and require byte-order discipline at every field boundary. Get one ntohs() wrong and you return garbage.

System Design

UDP Socket Loop

DNS runs over UDP. The server creates a single SOCK_DGRAM socket, binds to the configured port, and processes one query at a time in a blocking recvfrom() loop:

recvfrom() → dns_main() → check_in_config()
                              ↓ found         ↓ not found
                           send_dns()      foreign()

There is no select() or poll() — the server is single-client sequential. For an authoritative server handling low-volume internal queries, this is acceptable. For a production resolver under load, the bottleneck is the blocking foreign() call to the upstream DNS.

DNS Header and Wire Format

The DNS packet structure is defined with #pragma pack(push, 1) to prevent compiler padding — every field must align to the wire position exactly. The DNS_HEADER struct maps directly onto the first 12 bytes of every packet:

BitsField
16Query ID
1QR (query/response)
4Opcode
1AA (authoritative answer)
1TC (truncated)
1RD (recursion desired)
1RA (recursion available)
4Response code
16 × 4Question/Answer/Authority/Additional counts

The query name starts at buf[sizeof(DNS_HEADER)] — a label-encoded domain name in 3www6google3com format, null-terminated.

Label Decompression

DNS uses pointer compression to avoid repeating domain names. A byte with value ≥ 192 (binary 11000000) signals a compression pointer — the next byte gives the offset into the packet where the label continues.

ReadName() handles both raw labels and compressed pointers in a single pass. The compression offset calculation is:

offset = (*reader) * 256 + *(reader+1) - 49152

49152 is 0xC000 — the two-bit compression marker stripped from the 16-bit pointer field. When the function follows a pointer, it stops incrementing *count (the caller’s byte-advance counter), because the actual packet position didn’t move.

Two-Tier Lookup

Resolution follows a strict two-tier fallback:

  1. Local authority: check_in_config(name, qtype) searches zone files loaded from config.txt. Returns a vector<vector<string>> of matching resource records.
  2. Upstream forwarding: if no local records match, foreign() opens a new UDP socket, forwards the original query packet verbatim to the upstream resolver (e.g., 8.8.8.8), and proxies the response back to the client.

There is no caching between these tiers. Every miss causes a new upstream round-trip.

Binary RR Construction

Response packets are assembled byte-by-byte. The R_DATA struct is also #pragma pack(push, 1):

struct R_DATA {
    unsigned short type;
    unsigned short _class;
    int ttl;
    unsigned short data_len;
};

Each record type has its own construction function (A_type_RR, AAAA_type_RR, NS_type_RR, etc.) that allocates a RES_RECORD, fills the name via ChangetoDnsNameFormat, packs the rdata, and copies the assembled record into the send buffer via RR_copy_in_().

unsigned char* ReadName(unsigned char* reader, unsigned char* buffer, int* count) {
    unsigned char *name = (unsigned char*) malloc(256);
    unsigned int p = 0, jumped = 0, offset;

    name[0] = '\0';
    while (*reader != 0) {
        if (*reader >= 192) {
            // compression pointer: strip 0xC000, seek to offset
            offset = (*reader) * 256 + *(reader + 1) - 49152;
            reader = buffer + offset - 1;
            jumped = 1;
        } else {
            name[p++] = *reader;
        }
        reader++;
        if (jumped == 0) *count += 1;
    }
    name[p] = '\0';
    if (jumped == 1) *count += 1;

    // convert 3www6google3com → www.google.com
    for (int i = 0; i < (int)strlen((const char*)name); i++) {
        int p = name[i];
        for (int j = 0; j < p; j++) { name[i] = name[i+1]; i++; }
        name[i] = '.';
    }
    return name;
}

void foreign(int cs, char sbuf[], int slen, struct sockaddr_in csin) {
    char buf[65536] = {};
    struct sockaddr_in dest;

    int s = socket(AF_INET, SOCK_DGRAM, IPPROTO_UDP);
    dest.sin_family = AF_INET;
    dest.sin_port = htons(53);
    dest.sin_addr.s_addr = inet_addr(dns_servers[0]);

    sendto(s, sbuf, slen, 0, (struct sockaddr*)&dest, sizeof(dest));

    int i = sizeof(dest), rlen = 0;
    rlen = recvfrom(s, buf, 65536, 0, (struct sockaddr*)&dest, (socklen_t*)&i);

    // proxy response directly back to original client
    sendto(cs, buf, rlen, 0, (struct sockaddr*)&csin, sizeof(csin));
}

void A_type_RR(char* send_buf, int &stop, unsigned char name[], string ip, int ttl) {
    struct RES_RECORD *ans = new RES_RECORD;
    ans->name     = new unsigned char[64];
    ans->rdata    = new unsigned char[256];
    ans->resource = new R_DATA;

    ChangetoDnsNameFormat(ans->name, name);

    uint32_t ip_A = 0;
    inet_pton(AF_INET, ip.c_str(), (void*)&ip_A);
    memcpy(ans->rdata, &ip_A, sizeof(uint32_t));

    ans->resource->type     = htons(T_A);
    ans->resource->_class   = htons(1);
    ans->resource->ttl      = ntohl(ttl);
    ans->resource->data_len = htons(sizeof(uint32_t));

    RR_copy_in_(send_buf, stop, ans);
}

Takeaway

No caching means unbounded upstream load. Every authoritative miss opens a new UDP socket and blocks on recvfrom(). A TTL-respecting cache — even an unordered_map<string, CachedRecord> with expiry timestamps — would eliminate repeated upstream round-trips for popular domains and prevent the upstream resolver from becoming the bottleneck.

Manual memory management accumulates debt. Every A_type_RR call allocates three heap objects with new and never frees them. A single struct RES_RECORD allocated on the stack (or a smart pointer) would eliminate the leak. The same pattern repeats across all eight record type constructors.

ChangetoDnsNameFormat has no bounds checking. The function writes into a caller-supplied buffer with no length parameter. A domain name longer than the allocated 64 bytes overwrites adjacent memory. In a real implementation, this is the most likely source of a remotely exploitable buffer overflow.

The jump from this implementation to a production resolver requires two additions: a TTL-based response cache and a non-blocking upstream query model (either epoll or a thread pool) to handle concurrent misses without serializing all queries behind a single recvfrom().
