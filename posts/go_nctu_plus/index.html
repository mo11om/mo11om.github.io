The Problema

Building an authenticated web platform is straightforward when you own the identity system. NCTU Plus had a harder constraint: authentication must go through NYCU’s OAuth2 provider. The university controls user accounts, session validity, and profile data. Your API is a downstream consumer — you receive an authorization code, exchange it for a profile, and then issue your own session tokens.

This creates a three-party trust chain: browser → NYCU OAuth → your API. The engineering challenge is completing that chain cleanly: issuing short-lived JWTs, enforcing them on protected routes, supporting anonymous comment posting, and handling logout without a server-side session store.

System Design

Async Database Initialization

The server starts before the database connection is established:

router.Run(":8080")   ← starts immediately
go func() { database.DBconnect() }()   ← connects in background

The goroutine runs the GORM MySQL connection setup concurrently with the HTTP listener. This means the server is ready to accept connections within milliseconds — useful in containerized environments where the app container starts before the DB container is healthy.

The trade-off is explicit: if the database fails to connect, the server is running but every DB-touching handler will panic or return errors. A production system would add a readiness probe that blocks traffic until the connection pool is healthy.

Middleware Composition

Gin middleware is composed at the router level. Two middlewares wrap all routes:

MiddlewarePurpose
Cors()Dynamic origin reflection with credentials support
RequireAuth()JWT validation; injects user_id and student_id into context

The CORS middleware reflects the incoming Origin header rather than returning a wildcard *. This is required when Access-Control-Allow-Credentials: true is set — browsers reject credentialed cross-origin requests against wildcard origins.

JWT Lifecycle

The complete token lifecycle runs across three handlers:

  1. GET /oauth/login — redirects to NYCU’s OAuth authorize endpoint with client_id, scope=profile, and redirect_uri
  2. GET /oauth/code — receives the authorization code; exchanges it for an NYCU access token; fetches the user profile; issues a JWT; sets it as an httpOnly cookie
  3. GET /oauth/logout — sets the cookie with TTL -1 to expire it immediately

Tokens are signed with HS256 and carry user_id, student_id, and a 10-hour expiry. There is no refresh token — after expiry the user re-authenticates through NYCU OAuth.

Anonymous Comment Design

Comments support anonymous posting. The user_id foreign key is stored at write time, but the API does not expose it in public read endpoints. The DB layer decouples identity from content: GET /comments/course returns comment text, title, and timestamp — no author attribution unless the caller is authenticated and fetching their own comments via GET /comments/me.

func Cors() gin.HandlerFunc {
    return func(c *gin.Context) {
        origin := c.Request.Header.Get("Origin")
        if origin != "" {
            c.Header("Access-Control-Allow-Origin", origin)
            c.Header("Access-Control-Allow-Methods", "POST, GET, OPTIONS, PUT, DELETE, UPDATE, PATCH, TRACE")
            c.Header("Access-Control-Allow-Headers", "Origin, X-Requested-With, Content-Type, Accept, Authorization")
            c.Header("Access-Control-Allow-Credentials", "true")
        }
        if c.Request.Method == "OPTIONS" {
            c.AbortWithStatus(http.StatusNoContent)
        }
        c.Next()
    }
}

func Nycu_Oauth_Get_JWT(ctx *gin.Context) {
    code := ctx.Query("code")
    jwt_token, err := controllers.Get_jwt_token(code)
    if err != nil {
        ctx.Redirect(http.StatusTemporaryRedirect, os.Getenv("front_end_uri"))
        return
    }
    ctx.SetSameSite(http.SameSiteLaxMode)
    ctx.SetCookie("Authorization", jwt_token, 3600*24, "/", "", false, true)
    ctx.Redirect(http.StatusPermanentRedirect, os.Getenv("front_end_uri"))
}

func Nycu_delete_info(ctx *gin.Context) {
    ctx.SetSameSite(http.SameSiteLaxMode)
    ctx.SetCookie("Authorization", "", -1, "", "", false, true)
    ctx.Status(http.StatusOK)
}

func get_Jwt(profile profileResponse) (string, error) {
    expiresAt := time.Now().Add(10 * time.Hour).Unix()
    user, err := Check_user_exit_and_create(profile)
    if err != nil {
        return "", err
    }
    token := jwt.NewWithClaims(jwt.SigningMethodHS256, jwt.MapClaims{
        "user_id":    user.UserId,
        "student_id": profile.Username,
        "exp":        expiresAt,
    })
    return token.SignedString([]byte(os.Getenv("JWT_SECRET")))
}

Takeaway

Global DB is a hidden coupling point. The single database.Db global shared across all controller packages works in development but breaks under test — there is no seam to inject a test database. A proper refactor passes *gorm.DB as a parameter or uses a repository interface, enabling unit tests without a live MySQL instance.

N+1 queries are the silent tax. The FindCommentByTeacher() query joins three tables at the ORM level. GORM’s lazy association loading can generate one query per comment to fetch related records. Adding Preload() calls or rewriting as a single raw SQL join eliminates the problem — but only after you profile under load.

SameSite=Lax is the right default. Strict would break the OAuth redirect flow: the browser wouldn’t send the cookie on the cross-site redirect from NYCU back to the API. Lax allows top-level navigation cookies while blocking cross-site POST requests — the correct balance for this authentication pattern.
